# Model Selection Guide

Справочник: какую модель Claude использовать для каждой задачи в боте.
**Ссылаться при выборе модели, не спрашивать юзера.**

---

## Цены

| Модель | Вход / выход (за 1M токенов) |
|---|---|
| **Haiku 4.5** | $1 / $5 |
| **Sonnet 4.6** | $3 / $15 |
| **Opus 4.6** | $5 / $25 |

---

## Правило

```
Haiku  = служебные вызовы (роутинг, классификация, валидация, да/нет)
Sonnet = стандартная работа (чат, текст, код, анализ) ← DEFAULT
Opus   = глубокое рассуждение (стратегия, планирование, агентские сценарии, >200k контекст)
```

---

## Таблица по задачам бота

| Задача в боте | Модель | Почему |
|---|---|---|
| Router: классификация сообщения → enum | **Haiku** | Бинарный выбор, скорость |
| LLM relevance validation (entities relevant?) | **Haiku** | Да/нет ответ |
| Task completion parsing ("сделал portfolio") | **Haiku** | Определить какую задачу → CRUD |
| STEP 0: CLARIFY (уточняющие вопросы) | **Opus** | Нужно понять суть цели и задать ПРАВИЛЬНЫЕ вопросы |
| STEP 1: ASSESS (области знаний для цели) | **Opus** | Стратегическое решение: какие области ресёрчить. Ошибка = бесполезный ресёрч |
| STEP 2: RESEARCH (ресёрч пробелов) | **Sonnet** | Синтез web search + context, стандартная аналитика |
| STEP 3: DECOMPOSE (план + goals) | **Sonnet** | Структурирование, не глубокое рассуждение |
| Daily plan generation | **Sonnet** | Reasoning + контекст, Sonnet справляется |
| Evening summary | **Haiku** | Форматирование данных, не генерация |
| Drift check | **Sonnet** | Нужен анализ паттернов |
| Free chat | **Sonnet** | Разговор с контекстом |
| Interview prep | **Sonnet** | Синтез знаний + персонализация |
| Recruiter reply | **Sonnet** | Написание текста |
| Goal change challenge | **Opus** | Стратегическое решение, challenge с данными |
| Strategy pivot | **Opus** | Максимальная глубина анализа |

---

## Когда Opus обязателен

```
Задача определяет НАПРАВЛЕНИЕ (что ресёрчить, куда идти, менять ли стратегию)
  → Ошибка дорогая (бесполезный ресёрч, неправильная цель)
  → Opus

Задача исполнительная (написать текст, собрать план, ответить на вопрос)
  → Ошибка дешёвая (переделать)
  → Sonnet

Задача служебная (роутинг, yes/no, парсинг)
  → Ошибка тривиальная
  → Haiku
```
