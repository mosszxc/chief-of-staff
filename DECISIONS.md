# Architecture Decisions

Решения которые не очевидны из кода. Почему так, а не иначе.

---

### 001. Claude CLI subprocess, не Anthropic API
**Решение:** LLM вызовы через `claude --print -p` subprocess.
**Почему:** Подписка Claude Code уже оплачена → $0 сверху. Паттерн из Paperclip (claude-local adapter).
**Трейдоффы:** Startup time 3-8 сек per call. Нет кастомных tool definitions (решено через MCP).
**Дата:** 2026-04-05

### 002. YAML файлы, не SQLite/PostgreSQL
**Решение:** State хранится в YAML файлах (data/).
**Почему:** Один пользователь. YAML читаем человеком, легко дебажить. Нет overhead на БД.
**Трейдоффы:** Нет транзакций → asyncio.Lock per file. Нет запросов → читаем целиком.
**Когда пересмотреть:** Если перерастёт одного юзера (вероятно никогда).
**Дата:** 2026-04-05

### 003. Двухуровневый routing: Python + LLM
**Решение:** Кнопки/команды → Python напрямую (0 сек). Свободный текст → LLM routing.
**Почему:** Кнопка ✅ не требует AI. subprocess.run на каждое нажатие = 5-10 сек задержка.
**Трейдоффы:** Keyword matching может ошибиться → fallback на LLM.
**Дата:** 2026-04-05

### 004. MCP только для Claude Code sync, не для кнопок
**Решение:** Кнопки обновляют YAML напрямую (Python). MCP server — для Claude Code.
**Почему:** MCP = отдельный процесс при каждом Claude CLI вызове. Для кнопок — overkill.
**Трейдоффы:** Два пути записи в YAML (бот + MCP). asyncio.Lock защищает от race.
**Дата:** 2026-04-05

### 005. Compact plan + reasoning по кнопке
**Решение:** Утренний план — короткий (header + blocker + список задач). Reasoning — по кнопке [💡 Почему?].
**Почему:** Длинное reasoning утром в 08:00 = не дочитает → закроет → перестанет пользоваться (урок Motion).
**Дата:** 2026-04-05

### 006. Кнопки пронумерованы (1 ✅, 2 ✅)
**Решение:** Каждая строка кнопок начинается с номера задачи.
**Почему:** Без номеров непонятно какие кнопки к какой задаче (проблема из первого запуска).
**Дата:** 2026-04-05

### 007. data/ gitignored, SPEC.md приватный
**Решение:** Личные данные (YAML state, стратегия, SPEC) — только локально. GitHub = только код.
**Почему:** SPEC содержит реальные карьерные данные, финансы, военную обязанность.
**Дата:** 2026-04-05

### 008. Grimoire project "cos" для research knowledge
**Решение:** Результаты ресёрчей → pre-process → ingest в Grimoire (project: cos). Не сырые web results.
**Почему:** Raw web search = мусор в RAG. LLM extract facts → validate → metadata → ingest.
**Трейдоффы:** Extra LLM call для pre-processing. Оправдано: garbage in = garbage out.
**Дата:** 2026-04-05

### 009. Drift alert: рабочие дни, не календарные
**Решение:** "3 дня без прогресса" = 3 рабочих дня (сб/вс не считаются).
**Почему:** Ложный drift alert в выходные → раздражение → отключить уведомления → смерть продукта.
**Дата:** 2026-04-05

### 010. Timeout: 45s haiku, 120s sonnet
**Решение:** Увеличены с 30/60 до 45/120.
**Почему:** Claude CLI init = 3-8 сек. С MCP — ещё больше. 30 сек для sonnet = false timeout.
**Дата:** 2026-04-05
